# Crux Coach — Mobile Web QA Audit

Date: 2026-04-02
Tester: Burt
Environment tested: local app at http://127.0.0.1:3000
Viewport: 390x844 mobile portrait
Note: production `train.bkup.nyc` was returning 502 during this QA window, so this audit was performed against the local app. Browser gateway timed out before I could finish every single screen interaction, but I did get screenshots and code review across the core user flows.

## High-priority issues

### 1) Session detail action row is cramped / partially unusable on mobile
- Screen: planned session detail (`/plan/:id/session/:id`)
- Evidence: action buttons like Edit / Swap / Remove / Watch demo / Notes all collapse into a cramped single row in each exercise card.
- Impact: hard to tap, visually noisy, likely causes accidental taps.
- Fix direction: stack or wrap actions more intentionally on mobile; promote primary actions and move secondary actions into a compact menu or second row.

### 2) Session timer/work-rest timer controls are too dense for touch
- Screen: session detail
- Evidence: minute/second inputs + tiny Start buttons create very small touch targets.
- Impact: frustrating on phones, especially mid-workout.
- Fix direction: larger tap targets, cleaner vertical layout on small screens, possibly a single CTA per timer card.

### 3) Session header/back/status area is too compressed on mobile
- Screen: session detail
- Evidence: top bar packs back button, session meta, title, and status into one tight strip.
- Impact: reduced readability and poor hierarchy on the most important in-session screen.
- Fix direction: let the title wrap naturally, move status below header, simplify top layout for narrow widths.

### 4) Calendar monthly view is too dense for mobile
- Screen: `/calendar?view=monthly`
- Evidence: 7-column month grid remains full desktop-style on phone width.
- Impact: session cards are tiny, text is heavily compressed, tapping precise items will be annoying.
- Fix direction: create a truly mobile monthly treatment (agenda/list mode, week-strip + day agenda, or larger tappable day cells with less text).

### 5) Dashboard week strip is visually cramped and hard to scan
- Screen: `/dashboard`
- Evidence: the 7-day strip compresses session labels into tiny fragments on mobile.
- Impact: users cannot quickly understand the week plan at a glance.
- Fix direction: replace with horizontally scrollable day cards, fewer text lines, or show only session type + status.

### 6) Mobile nav/sidebar behavior likely needs hardening
- Screen: global mobile layout
- Evidence: layout uses an off-canvas sidebar with overlay, but no explicit body scroll lock / escape handling / stronger close affordances. During testing, the nav flow felt a bit brittle and one browser action auto-navigated unexpectedly while trying to interact.
- Impact: potential for awkward mobile navigation and trapped/scrolling background states.
- Fix direction: add body scroll lock when menu is open, better animation/state handling, and test open/close repeatedly.

## Medium-priority issues

### 7) Training Plans page cards are information-dense on mobile
- Screen: `/plans`
- Evidence: block cards contain status badge, dates, buttons, progress, nested weeks, and expandable subcontent all in one narrow column.
- Impact: readable, but feels crowded and a little overwhelming.
- Fix direction: tighten spacing hierarchy, reduce duplicated metadata, and make week sections more card-like / easier to skim.

### 8) Session detail exercise cards are too busy overall
- Screen: session detail
- Evidence: each exercise card contains title, multiple actions, description, sets, timers, notes, and editing affordances.
- Impact: scroll fatigue and cognitive overload on phone.
- Fix direction: progressive disclosure — keep the completion flow front-and-center, tuck edit/swap/notes behind expanders or menus.

### 9) Benchmarks rows are usable but cramped
- Screen: `/benchmarks`
- Evidence: long metric names wrap aggressively and value/date/unit layout is tight.
- Impact: harder to scan than it should be.
- Fix direction: simplify row layout on mobile, reduce line breaks, consider label above input/value meta below.

### 10) Edit Profile page is extremely long and visually repetitive
- Screen: `/profile/edit`
- Evidence: many checkbox groups and fields create a very long uninterrupted form.
- Impact: exhausting mobile editing experience.
- Fix direction: split into sections/cards with collapsible groups or a step-based edit flow; especially for strengths/weaknesses/disciplines/equipment.

### 11) Log Session CTA row is a little awkward
- Screen: `/log/new`
- Evidence: Parse Session and Save sit side-by-side with different visual weights.
- Impact: decent, but still easy to misread or tap the wrong thing on mobile.
- Fix direction: stronger hierarchy and spacing; potentially full-width stacked buttons on small screens.

## Lower-priority issues / polish

### 12) Inconsistent mobile density across screens
- Some screens feel nicely roomy (Progress, Coaches), while others feel packed (Session detail, Calendar, Plans).
- Fix direction: standardize mobile spacing scale and action density.

### 13) Button hit areas could be more consistently generous
- Several secondary action buttons look slightly undersized for workout-in-motion usage.
- Fix direction: ensure key mobile buttons meet ~44px tap target guidance.

### 14) More mobile-specific empty states would help
- Empty states that look fine on desktop could be more opinionated/helpful on phone.
- Fix direction: stronger guidance/CTA copy on empty weekly/monthly/calendar states.

## Previously known / still relevant items to keep in Codex scope
These came from earlier QA/docs and are still worth verifying/fixing if not already resolved:
- production 502 / timeout risk around long-running generation flows
- general loading/progress states for AI generation
- better mobile handling for dense form controls and checkboxes
- calendar/plan drag-drop behavior on touch devices may still need explicit mobile strategy (HTML5 drag/drop is fragile on mobile web)

## Recommended Codex task framing
1. Do a dedicated mobile-first pass on:
   - dashboard
   - plans
   - calendar (weekly + monthly)
   - session detail
   - profile edit
   - log session
   - benchmarks
2. Optimize for:
   - tap targets
   - visual hierarchy
   - reduced action clutter
   - smaller-screen readability
   - touch-friendly workout use
3. Preserve current dark-theme aesthetic.
4. After fixes: rebuild, run app locally, and browser-QA the same screens again at phone viewport.
